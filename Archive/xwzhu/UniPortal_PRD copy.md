# **UniPortal 统一门户平台 PRD**

## 产品愿景
UniPortal 旨在为WAGO SCADA、Asset Hub、EMS等工业软件APP提供**统一的用户入口、导航中心与协同工作台**，实现“一个系统、一致体验”的产品使用体验。通过统一的身份认证、权限管理、主题样式、多语言，消息中心和导航编排，Portal将原本分散的APP功能模块整合为业务连贯、体验一致的数字化工作环境，降低用户学习成本，提升跨系统协同效率，加速企业数字化转型的落地与推广。

---

## 背景
当前WAGO SCADA生态中已存在多个独立APP（如Asset Hub、EMS等），每个APP均有独立的登录入口、权限体系和用户界面。用户在跨系统操作时需频繁跳转、重复登录，体验割裂，且系统间的协同能力受限。

---

## 核心价值

### 1. 用户体验统一化，降低学习与操作成本
- 单点登录，一处认证：用户只需一次登录即可访问所有授权APP，告别重复输入账号密码的繁琐操作。
- 一致界面与交互：统一的视觉主题、多语言支持、表单规范与提示反馈，减少用户在不同APP间的适应成本。
- 智能导航与工作台：根据用户角色与权限动态呈现个性化菜单与快捷入口，支持自定义首页布局，提升操作效率。

### 2. 业务协同无缝化，打破系统孤岛
- 跨系统消息聚合：将SCADA告警、Asset Hub工单、EMS能效异常等消息统一归集至Portal消息中心，支持一键跳转处理。
- 统一待办任务管理：集中展示各系统待审批、待执行、待跟进的业务任务，支持批量处理与状态同步。
- 业务流程穿透：用户可在Portal内完成跨系统业务闭环，如从告警→工单创建→备件领用→维修记录，无需APP间手动跳转。

### 3. 系统集成标准化，降低开发与维护成本
- 模块化接入框架：提供标准化的APP注册、路由集成、权限声明接口，新APP接入成本大幅降低。
- 统一身份与权限治理：集中管理用户、角色、权限集，避免各APP重复开发权限模块，提升安全合规性。
- 配置驱动，灵活扩展：导航、菜单、主题等均可通过配置调整，支持动态启用/禁用APP，适应业务快速变化。

### 4. 生态扩展未来化，构建可持续平台架构
- 开放接入能力：为未来新增行业APP（如BMS、OEE、数字孪生、碳管理等）提供标准化接入路径。
- 开发者友好：提供完整的集成文档、API沙箱与调试工具，支持第三方ISV快速上架应用。
- 移动化与边缘协同：支持响应式布局与PWA技术，为移动巡检、边缘运维等场景提供统一入口。

### 5. 运维管理集中化，提升运营效率
- 统一监控与审计：集中查看各系统健康状态、用户操作日志、安全事件，便于合规审计与问题追溯。
- 用户自助服务：支持用户自主管理个人设置、查看操作指南、提交反馈，降低IT支持压力。
- 多租户与数据隔离：为企业集团、园区运营等场景提供租户隔离能力，支持分级管理。

## AI赋能：双模式智能应用生成

### AI-Developer模式：专业应用快速构建
**定位**：为开发者/实施顾问提供的AI增强开发工具
- **目标用户**：技术人员、系统集成商、实施团队
- **输出成果**：完整的、可二次开发的业务APP
- **典型场景**：
  - 构建“泵站智能巡检系统”完整APP
  - 开发“OEE综合监控平台”
  - 集成多个系统的“生产指挥中心”
- **价值**：将开发周期从**周/月级**缩短到**天级**

**使用流程**：
```
需求文档 → AI分析架构 → 代码生成 → 集成配置 → 完整应用
```

### AI-Copilot模式：业务看板自助生成
**定位**：为业务人员提供的零代码页面生成器
- **目标用户**：业务主管、运营人员、一线员工
- **输出成果**：可直接使用的交互式看板/页面
- **典型场景**：
  - 快速创建“今日设备故障统计看板”
  - 生成“各车间能耗趋势对比页面”
  - 制作“待处理工单优先级排序视图”
- **价值**：将应用创建从**需要IT支持**变为**自助完成**，响应时间从几天缩短到几分钟

**使用流程**：
```
自然语言描述 → AI立即生成 → 微调使用
（例：“显示今天振动超标的设备”）
```

## 典型客户场景举例

### 场景一：大型水务集团运营中心
**客户背景**：某水务集团已部署WAGO SCADA监控各水厂运行，使用Asset Hub管理水泵、阀门等设备，EMS监控能耗，但运维人员需在3个APP间频繁切换。

**使用UniPortal后的体验**：
1. **统一登录**：调度员小王早晨登录UniPortal一次，即可看到SCADA监控画面、设备健康状态、能耗报表三个模块
2. **智能告警处理**：系统弹出告警：“A泵站#3水泵振动异常”。小王在UniPortal消息中心直接点击告警，系统自动跳转至Asset Hub工单创建页面，设备信息、测点数据已自动填充
3. **跨APP协同**：创建工单后，UniPortal自动同步至维修班组移动端，同时关联EMS中的能耗异常记录
4. **AI快速看板**：运维经理通过AI-Copilot描述需求：“显示本周设备故障率最高的三个车间”，立即生成分析看板

### 场景二：智能制造企业数字化转型
**客户背景**：某汽车零部件制造企业，生产线使用SCADA监控，计划部使用ERP，设备部使用自研维护系统，能源部使用EMS，各部门数据割裂。

**使用UniPortal后的效果**：
1. **角色化门户**：
   - 生产主管：登录后看到SCADA实时数据、OEE看板、设备状态
   - 设备经理：看到Asset Hub设备树、预防性维护计划、备件库存
   - 能源主管：看到EMS能耗分析、碳排放数据、节能项目进度
2. **AI-Developer应用**：IT部门使用AI-Developer模式，2天内构建了“生产质量追溯系统”，集成MES、SCADA、ERP数据
3. **AI-Copilot看板**：质量主管用自然语言生成“批次不良率波动分析看板”，实时监控质量趋势

### 场景三：园区综合服务商
**客户背景**：某产业园区运营商，需管理多栋楼宇的BMS系统、公共区域能源系统、入驻企业服务系统等多个独立平台。

**UniPortal带来的价值**：
1. **多租户隔离**：为不同入驻企业提供专属门户界面，仅显示其租用区域的监控数据和服务入口
2. **综合运维中心**：园区运维团队在一个界面中同时监控：
   - 各楼宇空调、照明运行状态（来自BMS）
   - 公共区域能耗数据（来自EMS）
   - 设备维修进度（来自Asset Hub）
3. **快速响应服务**：入驻企业通过UniPortal提交维修申请→自动分派至运维团队→完成后企业在线评价

### 场景四：集成商交付标准化
**客户背景**：自动化集成商为不同行业客户提供定制化解决方案，每个项目需整合SCADA、设备管理、能效分析等多个系统。

**UniPortal的交付优势**：
1. **标准化门户框架**：每个项目均以UniPortal为统一入口，客户体验一致
2. **快速集成**：新开发的APP通过标准接口接入UniPortal，无需重复开发权限、导航、消息等基础功能
3. **AI加速开发**：使用AI-Developer模式，将定制化APP开发时间缩短70%
4. **降低培训成本**：客户员工只需学习一次UniPortal操作，新增APP模块无学习门槛
5. **灵活授权**：可根据客户预算和需求，在UniPortal中按模块启用/禁用功能

## 市场定位
### 产品定位
UniPortal统一门户平台是面向WAGO SCADA生态的**统一用户入口与协同工作台**，适用于已部署或计划部署多套工业软件系统的中大型企业。

### 目标客户
- 已使用WAGO SCADA + Asset Hub + EMS等多系统的客户，希望提升使用体验与协同效率；
- 计划构建“SCADA + 上层APP”生态的集成商与ISV，需要一个统一的交付入口；
- 企业IT部门，希望通过统一门户整合多个业务系统，降低运维与培训成本。

### 核心价值总结
- **体验统一**：消除APP跳转与重复登录，提升用户满意度；
- **效率提升**：通过消息聚合、待办统一、快捷导航，加速业务闭环；
- **生态整合**：为WAGO SCADA生态提供可扩展的接入框架，支持未来APP无缝集成；
- **智能赋能**：AI双模式支持，既加速专业开发，又赋能业务创新；
- **降本增效**：减少各系统独立开发登录、权限、导航等基础模块的成本。

**一句话总结**：UniPortal是连接WAGO SCADA生态各APP的智能门户，让用户在一个平台上完成所有业务操作，并通过AI能力让每个人都能快速创建所需应用。
