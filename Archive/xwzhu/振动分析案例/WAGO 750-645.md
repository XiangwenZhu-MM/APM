## 1. 硬件接口：IEPE 传感器的“翻译官”

大多数工业级加速度传感器都采用 **IEPE（集成电路压电式）** 标准。这种传感器的特点是需要一个恒定的直流电流供电，并同时在同根线上返回交流电压信号。

* **通道数**：750-649 拥有 **4 个通道**。
* **供电能力**：模块内部集成了 4mA 的恒流源（24V DC），直接驱动传感器，无需额外配电器。
* **测量范围**：支持高达 **25kHz** 的采样频率，这对于捕捉轴承磨损产生的高频冲击振动至关重要。

## 2. 核心功能：它在模块内部做了什么？

在数据上传到 CPU 之前，750-649 可以在内部实时完成以下计算，这极大地减轻了控制器的负担：

1. **有效值（RMS）计算**：反映振动的整体能量，常用于判断设备是否“整体抖动过大”。
2. **峰值（Peak/Peak-to-Peak）**：捕捉瞬间的撞击，比如齿轮断齿或严重的轴承剥落。
3. **波峰因子（Crest Factor）**：这是判断轴承初期磨损的黄金指标（）。
4. **积分运算**：可以将加速度信号自动积分转换为**振动速度**（mm/s），符合 ISO 10816 振动标准。

## 3. 边缘侧 FFT 变换的实现方案

正如你之前设想的，你不想持续上传原始波形（那会产生巨大的流量）。在 WAGO 架构中，FFT 是这样分工的：

### 第一阶段：信号预处理（750-649 模块层）

模块持续监测振动的 RMS 和 Peak 值。你可以设定一个**阈值**（例如：振动速度超过 4.5mm/s）。

### 第二阶段：特征提取（PFC200/Edge Controller 层）

当模块检测到数值异常时，触发控制器执行以下逻辑：

* **捕获快照**：控制器通过内部总线读取一段高频原始采样序列（Time Domain Data）。
* **FFT 计算**：在控制器内调用 **WAGO Vibration Monitoring 库** 或运行 **Docker 容器中的 Python (NumPy/SciPy)** 脚本。
* **频谱分析**：
* 识别 **BPFO**（外圈故障频率）、**BPFI**（内圈故障频率）等特征点。
* 如果特征频率处的幅值显著增加，则确认为轴承损伤。

## 5. 部署建议

如果你要落地这个项目，建议配置如下：

1. **硬件**：1个 PFC200 控制器 (750-8212) + 1个振动模块 (750-649)。
2. **传感器**：选择灵敏度为 100mV/g 的标准工业加速度计。
3. **软件**：下载 WAGO 提供的 **Vibration Monitoring Library**。它已经封装好了 FFT 和标准过滤器的功能块，你只需要输入轴承的转速、滚珠数量等参数，它就能自动算出故障频率。
